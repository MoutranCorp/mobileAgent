# App assets

## `bootstrap-<arch>.tar.gz` (generated — not committed)

The app bundles a **Termux bootstrap** tarball here. It's large and
architecture-specific, so it's **not committed** (see `.gitignore`) — build it
once with the committed recipe:

```bash
ARCH=aarch64 bash provisioning/make-bootstrap.sh   # writes bootstrap-aarch64.tar.gz here
```

[`provisioning/make-bootstrap.sh`](../../../../../provisioning/make-bootstrap.sh)
downloads the official `termux/termux-packages` bootstrap zip, recreates the
symlinks it lists in `SYMLINKS.txt` (zip can't store symlinks), and repacks it as
a GZIP tar (the on-device toybox `tar` can't do `.xz`/`.zst`). This is the **base**
Termux userland (apt/pkg); `proot-distro` + Debian + the toolchain are installed
by `setup-guest.sh` on first launch (needs network, one-time — the auto-provision
route).

`BootstrapManager.extractBootstrap()` extracts it into the app's private
`files/runtime/usr`; `targetSdk 28` is what allows the extracted binaries to be
`exec()`-ed from the data dir. Because the Termux binaries hardcode the Termux
prefix (`/data/data/com.termux/files/usr`) and can't be patched in place (our
prefix is a different length), `RuntimeLauncher` runs them under **termux-exec**
(`LD_PRELOAD=…/libtermux-exec-ld-preload.so` + `TERMUX__PREFIX`/`TERMUX__ROOTFS`),
which the bootstrap already ships — it rewrites those paths at runtime.

> **aapt2 gunzips `.gz` assets at package time and strips the suffix**, so what we
> build as `bootstrap-aarch64.tar.gz` ships in the APK as `bootstrap-aarch64.tar`
> (plain). The code matches by prefix / adapts the `tar` flags accordingly — don't
> "fix" the names to add `.gz` back.

If no bootstrap is bundled the app falls back to **external-broker mode**: run the
broker on a computer + `adb reverse tcp:8765 tcp:8765`, or `--engine mock`.

## `broker.tar.gz` (generated by Gradle — not committed)

The Gradle `stageBroker` task tars `../broker` (source only; `node_modules`
installed on-device) into here so the app delivers the broker into the guest with
no separate clone. Also gunzipped to `broker.tar` by aapt — handled the same way.

## `scripts/`

On-device shell scripts copied into `files/runtime/scripts` on first run:

- `setup-guest.sh` — installs proot-distro + Debian + Node toolchain + Claude CLI,
  and **delivers the bundled broker** into `~/agent-broker` (bind-mounted in).
- `run-broker.sh`  — starts the broker inside the guest.
